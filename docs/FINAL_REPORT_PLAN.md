## Key Design Decisions

- **Personal model (5-class) is the primary subject**, not the HMDB model (3-class).
  The personal model is what runs in the game. The HMDB model was a stepping stone.

- **`ActionLSTM = ActionTCN` alias tells a story.** The project planned to use LSTM,
  switched to TCN during implementation because it trains faster on small data and
  has no hidden state to manage during inference. The blog should explain this honestly.

- **Window length is the most meaningful hyperparameter** for this project. Shorter
  windows (10 frames) mean faster response but less temporal context. Longer windows
  (30 frames) are smoother but add latency. This is a real tradeoff in a real-time game.

- **The notebook must be self-contained.** No webcam required. All figures generated
  from saved data files and model checkpoints.

- **The blog must render locally.** No CDN, no external fonts, no JavaScript frameworks.
  Just HTML + CSS + local images.
